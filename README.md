# CS340-Module-8-Journal

How do you write programs that are maintainable, readable, and adaptable? Especially consider your work on the CRUD Python module from Project One, which you used to connect the dashboard widgets to the database in Project Two. What were the advantages of working in this way? How else could you use this CRUD Python module in the future?

When I write programs, I focus on keeping my code organized, readable, and easy to reuse later. In this project, I built a CRUD Python module that handled all interactions with the MongoDB database, and then reused that same module in my dashboard to power the data table, filters, and visualizations. Instead of writing database queries directly inside the dashboard code, I separated that logic into its own class with create, read, update, and delete methods. This made my code much cleaner and easier to manage because I always knew where the database logic lived. It also made debugging simpler since I could test database functionality separately from the dashboard. One big advantage of working this way is reusability. I can take this same CRUD module and use it in future projects like a web app, API, or even another dashboard without rewriting everything. I could also expand it by adding more advanced queries or connecting it to a different dataset.


How do you approach a problem as a computer scientist? Consider how you approached the database or dashboard requirements that Grazioso Salvare requested. How did your approach to this project differ from previous assignments in other courses? What techniques or strategies would you use in the future to create databases to meet other client requests?

When I approach a problem as a computer scientist, I start by breaking it down into smaller pieces and understanding what the system needs to do as a whole. For this project, I focused on how Grazioso Salvare needed to filter and identify rescue dogs based on specific criteria, and then designed my solution around that. I built the database connection first, then created the CRUD module, and finally developed the dashboard interface using Dash. This approach was different from earlier assignments because it felt more like a real world system where everything had to work together, not just a single piece of code. I also had to think about how users would interact with the system, such as using filters, viewing charts, and selecting rows to update the map. The MVC structure I used, where MongoDB acts as the model, the CRUD module and callbacks act as the controller, and the dashboard UI acts as the view, helped me organize everything clearly. In the future, I would continue using this approach by designing the data structure first, then building reusable logic, and finally creating a user friendly interface.


What do computer scientists do, and why does it matter? How would your work on this type of project help a company, like Grazioso Salvare, to do their work better?

Computer scientists build systems that help people work with data more efficiently, and that is exactly what this project does. My dashboard allows Grazioso Salvare to quickly filter through the Austin Animal Center dataset and identify dogs that meet the requirements for different types of rescue training. Instead of manually searching through thousands of records, they can use interactive filters, visual charts, and a map to make decisions faster and more accurately. This kind of system reduces human error and improves productivity. It shows how software can take a large amount of raw data and turn it into something meaningful and easy to use, which is valuable for any organization trying to make data driven decisions.
